# 쉬운 설명 대본

이 대본은 대시보드 프로젝트를 처음 접하는 사람에게 설명할 때 사용하세요.

## 프로젝트 한 줄 설명
"ROS2 로봇 시스템이 지금 제대로 움직이고 있는지, 데이터는 잘 통하는지 웹에서 실시간으로 확인하는 대시보드입니다."

## 전체 구조 설명
"로봇 내부의 상태를 가져오는 '백엔드'와 그 데이터를 웹 브라우저에서 예쁘게 그려주는 '프론트엔드'로 나뉘어 있습니다."

## 기능 설명
- **Topic**: 로봇이 주고받는 데이터 채널입니다. 여기선 어떤 데이터가 몇 개나 흐르는지 보여줍니다.
- **Service/Action**: 로봇에게 일을 시키는 명령 체계입니다. 성공했는지, 응답은 빠른지 보여줍니다.
- **Node**: 로봇의 두뇌들입니다. 어떤 두뇌가 살아있고 어떤 게 죽었는지 확인합니다.
- **Alert**: 로봇에 문제가 생기면 알려주는 알람입니다.

## 안전 설계
"백엔드는 로봇 시스템에 부하를 주지 않기 위해 꼭 필요한 정보만 가져오고, 서비스 호출도 등록된 안전한 기능만 수행합니다."

## 시연 순서
1. "먼저 로봇(ROS2) 시스템을 켭니다."
2. "대시보드를 엽니다."
3. "토픽 목록에서 데이터 흐름을 보여줍니다."
4. "서비스가 정상 작동하는지 보여줍니다."
5. "문제가 생기면 알람이 뜨는 걸 보여줍니다."

---

### 내가 반드시 알아야 할 3줄 요약
1. 이 프로젝트는 ROS2 로봇 시스템을 쉽게 모니터링하기 위한 웹 대시보드입니다.
2. 시스템 부하를 방지하는 안전한 모니터링 방식이 설계되어 있습니다.
3. Node, Topic, Service, Action 단위로 로봇의 상태를 시각화합니다.
