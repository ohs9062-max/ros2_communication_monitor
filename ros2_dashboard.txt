ROS2 Communication Monitor Dashboard

1. 프로젝트 개요
ROS2 기반 시스템의 Node, Topic, Service, Action 상태를 실시간으로 모니터링하고
웹 대시보드에서 통합 관리할 수 있는 시스템을 개발한다.
기존에는 ros2 topic, ros2 service, ros2 action 명령어를 각각 실행하여
상태를 확인해야 했으나, 본 프로젝트는 이를 하나의 화면에서 실시간으로
확인할 수 있도록 구현한다.

2. 시스템 구성
ROS2 Nodes
├── Topic
├── Service
└── Action
    ↓
ROS2 Monitor Node
    ↓
REST API / WebSocket
    ↓
Web Dashboard

3. 주요 기능

3.1 Topic 모니터링
- Topic 목록 조회
- Publisher / Subscriber 정보 표시
- 메시지 수신 주기(Hz) 표시
- Topic 연결 상태 확인
- 메시지 미수신 시 경고 표시

3.2 Service 모니터링
- Service 목록 조회
- Service Server 상태 확인
- Service 호출 결과 표시
- 응답 시간(Response Time) 측정
- 응답 실패 및 Timeout 알림

3.3 Action 모니터링
- Action 목록 조회
- Goal 상태 확인
- Feedback 및 Result 표시
- 실행 시간 측정
- 실패 및 취소 상태 표시

3.4 Node 모니터링
- 실행 중인 Node 목록 조회
- Node 실행 상태 표시
- Publish / Subscribe 정보 확인
- 비정상 종료 감지

3.5 이상 상태 알림
다음과 같은 이상 상황 발생 시 Dashboard에 경고를 표시한다.
- Topic 데이터 미수신
- Node 종료
- Service 응답 실패
- Action 실행 실패
- 통신 지연

4. Dashboard 화면
Dashboard에서는 다음 정보를 실시간으로 제공한다.
- Node 상태
- Topic 상태 및 Hz
- Service 상태
- Action 진행 상태
- 시스템 경고 메시지
- ROS2 통신 구조 시각화
